آشنایی با شاخه‌ها (Branch) در Git

در گیت، شاخه یا همان Branch یکی از مهم‌ترین ابزارها برای مدیریت کد و همکاری روی پروژه‌هاست. شاخه‌ها به ما اجازه می‌ده بدون اینکه روی نسخه اصلی یا شاخه اصلی پروژه (معمولاً main یا master) تاثیر بگذاریم، تغییرات جدید ایجاد کنیم.

شاخه چیست؟

شاخه در واقع یک مسیر جداگانه برای توسعه‌ی پروژه است. وقتی شاخه جدیدی می‌سازیم، عملاً از نقطه‌ای مشخص از پروژه کپی می‌گیریم و می‌توانیم روی آن تغییرات انجام دهیم بدون اینکه به شاخه اصلی پروژه آسیبی برسد.

چرا شاخه‌ها مهم هستند؟

  • جلوگیری از بهم‌ریختگی کد اصلی: هر کد جدید یا ویژگی تازه روی یک شاخه جداگانه توسعه داده می‌شود تا اگر مشکلی داشت، کد اصلی سالم باقی بماند.
  • کار تیمی آسان‌تر: هر عضو تیم روی شاخه خودش کار می‌کند و وقتی همه چیز درست بود، تغییرات را به شاخه اصلی اضافه می‌کند.
  • امکان آزمایش و توسعه موازی: می‌توان چندین ویژگی یا باگ را همزمان روی شاخه‌های مختلف پیگیری کرد.

ساخت و مدیریت شاخه‌ها

ساخت شاخه جدید

git checkout -b <نام-شاخه>

مثلاً:

git checkout -b feature/login

رفتن به یک شاخه موجود

git checkout <نام-شاخه>

مثلاً:

git checkout main

دیدن لیست شاخه‌ها

git branch

شاخه‌ای که در حال حاضر روی آن هستید با علامت * مشخص می‌شود.

ادغام شاخه‌ها (Merge)

بعد از اینکه روی شاخه جدید تغییرات لازم را انجام دادید و همه چیز درست بود، می‌توانید آن را به شاخه اصلی ادغام کنید:

  1. ابتدا به شاخه اصلی برمی‌گردید:
git checkout main
  1. سپس شاخه تغییرات را با شاخه اصلی ادغام می‌کنید:
git merge <نام-شاخه>

مثلاً:

git merge feature/login

حذف شاخه

git branch -d <نام-شاخه>

نکات مهم

  • قبل از ادغام شاخه‌ها، بهتر است تغییرات شاخه اصلی را بروز کنید (pull کنید) تا تداخل (conflict) کمتری داشته باشید.
  • اگر در زمان ادغام، تغییرات دو شاخه با هم تداخل داشتند، باید به صورت دستی تداخل‌ها را حل کنید.
  • استفاده درست از شاخه‌ها باعث می‌شود پروژه مرتب، قابل مدیریت و قابل توسعه باشد.
